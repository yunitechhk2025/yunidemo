# 🚀 所有部署方式汇总

本文档列出所有可用的部署方式，从最简单到最复杂。

---

## 📊 快速对比

| 部署方式 | 难度 | 成本 | 推荐度 | 适用场景 |
|---------|------|------|--------|---------|
| **Vercel** | ⭐ 最简单 | 免费/付费 | ⭐⭐⭐⭐⭐ | 快速上线，个人项目 |
| **Railway** | ⭐⭐ 简单 | 免费/付费 | ⭐⭐⭐⭐⭐ | 一键部署，自动CI/CD |
| **Netlify** | ⭐⭐ 简单 | 免费/付费 | ⭐⭐⭐⭐ | 前端+API，静态站点 |
| **Render** | ⭐⭐ 简单 | 免费/付费 | ⭐⭐⭐⭐ | 类似Heroku，简单易用 |
| **Google Cloud Run** | ⭐⭐⭐ 中等 | 按量付费 | ⭐⭐⭐⭐ | 企业级，可扩展 |
| **Docker + 服务器** | ⭐⭐⭐ 中等 | 服务器费用 | ⭐⭐⭐ | 自有服务器 |
| **PM2 + VPS** | ⭐⭐⭐⭐ 较难 | 服务器费用 | ⭐⭐⭐ | 传统部署方式 |

---

## 🌟 方案一：Vercel（最简单，推荐）

### 特点
- ✅ 完全免费（个人项目）
- ✅ 自动 HTTPS
- ✅ 全球 CDN
- ✅ 零配置部署
- ✅ 自动 CI/CD

### 部署步骤

```bash
# 1. 安装 Vercel CLI
npm install -g vercel

# 2. 登录
vercel login

# 3. 部署
vercel --prod
```

### 或使用网页界面
1. 访问 https://vercel.com
2. 连接 GitHub/GitLab/Bitbucket
3. 导入项目
4. 自动部署

**配置文件已准备好：`vercel.json`**

---

## 🚂 方案二：Railway（一键部署）

### 特点
- ✅ 免费额度：$5/月
- ✅ 自动检测 Node.js
- ✅ 自动部署
- ✅ 简单易用

### 部署步骤

1. **访问 Railway**
   - 打开 https://railway.app
   - 使用 GitHub 登录

2. **创建项目**
   - 点击 "New Project"
   - 选择 "Deploy from GitHub repo"
   - 选择您的仓库

3. **自动部署**
   - Railway 自动检测 Node.js
   - 自动构建和部署
   - 生成访问 URL

4. **设置环境变量**（可选）
   - 在项目设置中添加环境变量

---

## 🌐 方案三：Netlify

### 特点
- ✅ 免费额度充足
- ✅ 自动 HTTPS
- ✅ 全球 CDN
- ✅ 支持 Serverless Functions

### 部署步骤

```bash
# 1. 安装 Netlify CLI
npm install -g netlify-cli

# 2. 登录
netlify login

# 3. 部署
netlify deploy --prod
```

### 或使用网页界面
1. 访问 https://netlify.com
2. 拖拽项目文件夹
3. 自动部署

---

## 🎨 方案四：Render

### 特点
- ✅ 免费额度：750小时/月
- ✅ 自动 HTTPS
- ✅ 零配置
- ✅ 类似 Heroku

### 部署步骤

1. **访问 Render**
   - 打开 https://render.com
   - 注册账号

2. **创建 Web Service**
   - 点击 "New +" > "Web Service"
   - 连接 GitHub 仓库

3. **配置**
   - Build Command: `npm install`
   - Start Command: `node src/api.js`
   - Environment: `Node`

4. **部署**
   - 点击 "Create Web Service"
   - 自动部署

## ☁️ 方案五：Google Cloud Run（已准备）

### 特点
- ✅ 按量付费
- ✅ 自动扩缩容
- ✅ 全球部署

### 部署命令
```bash
gcloud run deploy my-demo \
  --source . \
  --region asia-east1 \
  --allow-unauthenticated
```

**详细文档：`GCLOUD-DEPLOY.md`**

## 🐳 方案六：Docker + 任意服务器

### 特点
- ✅ 环境一致
- ✅ 易于迁移
- ✅ 支持任何云服务商

### 部署步骤

```bash
# 1. 构建镜像
docker build -t my-demo .

# 2. 运行容器
docker run -d -p 3000:3000 --name my-demo my-demo

# 或使用 docker-compose
docker-compose up -d
```

**配置文件：`Dockerfile`, `docker-compose.yml`**

---

## 🔧 方案七：PM2 + VPS/云服务器

### 特点
- ✅ 完全控制
- ✅ 适合长期运行
- ✅ 需要服务器

### 部署步骤

```bash
# 1. 安装依赖
npm install --production

# 2. 启动服务
pm2 start ecosystem.config.js --env production

# 3. 设置开机自启
pm2 save
pm2 startup
```

**配置文件：`ecosystem.config.js`**

---

## 🎯 方案八：Heroku

### 特点
- ✅ 简单易用
- ✅ 免费额度（已取消）
- ✅ 需要付费

### 部署步骤

```bash
# 1. 安装 Heroku CLI
# 访问 https://devcenter.heroku.com/articles/heroku-cli

# 2. 登录
heroku login

# 3. 创建应用
heroku create your-app-name

# 4. 部署
git push heroku main
```
