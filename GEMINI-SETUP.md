# 🤖 Gemini AI 配置指南

## 为什么需要 Gemini？

集成 Gemini AI 后，系统可以：
1. **智能理解**：理解用户的自然语言描述，无需精确关键词
2. **自动映射**：将任何痛点描述自动映射到最合适的行业
3. **个性化建议**：根据用户具体情况生成定制化方案建议
4. **兜底机制**：如果 AI 不可用，自动降级到关键词匹配

---

## 🚀 快速配置（3步）

### 步骤 1：获取 Gemini API Key

1. 访问：https://makersuite.google.com/app/apikey
2. 使用 Google 账号登录
3. 点击 "Create API Key"
4. 复制生成的 API Key

### 步骤 2：配置环境变量

打开项目根目录的 `.env` 文件，替换 API Key：

```env
GEMINI_API_KEY=你的_API_Key_粘贴在这里
```

### 步骤 3：重启服务器

```bash
# 停止当前服务器（Ctrl + C）
# 重新启动
npm start
```

看到 `✅ Gemini AI 已启用` 表示配置成功！

## 🔍 工作原理

### 1. AI 分析流程
```
用户输入 → Gemini 分析 → 映射到行业 → 返回解决方案 → 生成个性化建议
```

### 2. 降级机制
```
如果 Gemini 不可用 → 自动使用关键词匹配 → 确保系统始终可用
```

### 3. 返回信息
- **行业映射**：AI 选择的行业 + 置信度
- **分析理由**：为什么选择这个行业
- **个性化建议**：针对用户痛点的定制建议

## ⚠️ 常见问题

### Q1: 看到 "未配置 GEMINI_API_KEY" 警告
**A:** 说明 `.env` 文件中的 API Key 还是默认值，请按步骤 1-2 配置

### Q2: API 调用失败
**A:** 可能原因：
- API Key 错误或过期
- 网络问题（需要访问 Google 服务）
- 超出免费配额

系统会自动降级到关键词匹配，不影响使用

### Q3: 不想使用 AI，只用关键词匹配
**A:** 不配置 API Key 即可，系统会自动使用关键词匹配模式

### Q4: 响应速度慢
**A:** AI 分析需要 1-3 秒，这是正常的。可以：
- 添加加载动画提示用户等待
- 使用缓存机制减少重复请求

## 🎯 AI vs 关键词匹配对比

| 特性 | AI 模式 | 关键词模式 |
|------|---------|-----------|
| 理解能力 | 理解语义和上下文 | 只匹配关键词 |
| 准确度 | 高（智能分析） | 中（依赖关键词） |
| 灵活性 | 支持任意描述 | 需要包含关键词 |
| 响应速度 | 1-3秒 | <100ms |
| 成本 | 免费额度内免费 | 完全免费 |
| 个性化 | 生成定制建议 | 固定模板 |

## 🔧 高级配置

### 自定义 AI 模型
编辑 `src/gemini-service.js`：

```javascript
// 使用更快的模型
this.model = this.genAI.getGenerativeModel({ model: 'gemini-pro' });

// 或使用更强大的模型（如果可用）
this.model = this.genAI.getGenerativeModel({ model: 'gemini-1.5-pro' });
```

### 调整 AI 提示词
修改 `src/gemini-service.js` 中的 `prompt` 变量，可以：
- 调整分析风格
- 增加行业知识
- 优化输出格式
